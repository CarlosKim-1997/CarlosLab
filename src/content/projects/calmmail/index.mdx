## 무엇을 하나요

Gmail inbox를 백그라운드에서 보고, VIP·답장 대기·긴급 스레드만 표면화합니다. 아침 브리핑은 요청할 때만 만듭니다.

## 왜 만들었나요

매 메일마다 AI가 개입하는 것보다, 규칙으로 걸러 내고 필요할 때만 AI를 쓰고 싶었습니다.

## 어떻게 동작하나요

Electron 메인에서 Gmail OAuth → 폴링 → 규칙 엔진 → (요청 시) AI 브리핑. AI는 메모리를 직접 바꾸지 않고 제안만 반환합니다.

## 설계 메모

- 본문 미저장: 메타데이터 + 스니펫(280자)만 SQLite
- gmail / monitor / rules / memory / ai 등 모듈 분리
- CalmMail 관리 llama.cpp + Apache 2.0 GGUF 모델

## 어려웠던 점

AI가 Gmail을 대신 조작하지 않도록 스코프·검증·호출 예산을 코드로 막는 것
