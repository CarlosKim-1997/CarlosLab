## 무엇을 하나요

고객 이메일 한 통이 들어오면 AI가 의도·담당 부서·우선순위를 분석하고, 비즈니스 규칙에 맞는 워크플로우를 생성한 뒤 Jira 티켓·Slack 알림·이메일 회신까지 자동으로 실행합니다.

## 왜 만들었나요

대기업 AI 전환 PoC에서 “분류 2시간 → 5분” 같은 설득력 있는 시연을 한 화면으로 보여주고 싶었습니다.

## 어떻게 동작하나요

Streamlit UI → AI Analysis Engine → Workflow Generator → Automation Engine(플러그인 레지스트리) 순으로 처리됩니다. 새 시스템은 `@register_integration` 데코레이터만으로 추가할 수 있습니다.

## 기술적 결정

- **개방-폐쇄 원칙**: SAP·ServiceNow 같은 확장은 기존 엔진 코드 수정 없이 플러그인으로
- **시뮬레이션 모드**: 실제 API 키 없이도 데모 가능
- **JSON 구조화 출력**: LLM 결과를 바로 downstream 시스템에 연결

## 어려웠던 점

컨설팅 데모와 실제 연동 사이의 균형 — “보이는 것”과 “동작하는 것”을 동시에 만족시키는 UX 설계

## 배운 점

엔터프라이즈 AI PoC는 모델 성능보다 **워크플로우 가시성**과 **시스템 연동 스토리**가 더 중요할 때가 많습니다.
